Choix de la configuration.

LODEL a besoin d'accéder à des scripts dans un répertoire inclus automatiquement ou dont le chemin absolu est connu et fixe.

Option 1 et 2

Si le chemin d'accès absolu à LODEL est connu et fixe préciser ce chemin ci-dessous.
Sinon, LODEL a besoin d'un répertoire inclus automatiquement (variable de configuration "include_path" de php). Si c'est le cas, choisir la deuxième option et préciser le chemin relatif vers ce répertoire.

Erreur: Impossible d'accéder aux scripts par le répertoire . Erreur: Le répertoire "" n'existe pas et il n'est pas possible de le créer.

Option 1

Verifier le chemin entré ci-dessous. Sinon vérifier les droits d'écriture dans le répertoire parent du répertoire "".

Option 2

Créer manuellement le répertoire "" et y copier les fichiers du répertoire lodel/scripts. Positionner les droits pour que le serveur puisse accéder aux fichiers en lecture.

Erreur: Impossible de copier les scripts dans le répertoire .

Option 1

Verifier le chemin entré ci-dessous. Sinon vérifier les droits d'écriture dans le répertoire parent du répertoire "".

Option 2

Copier les fichiers du répertoire lodel/scripts dans le répertoire"". Positionner les droits pour que le serveur puisse accéder aux fichiers en lecture.

Erreur: L'inclusion automatique dans le répertoire ne fonctionne pas. Vérifier le répertoire entré ci-dessous ainsi que la configuration de php. Si le problème persiste... il n'y a pas grand chose à faire. Attention: Le chemin indiqué ci-dessous a été "deviné" par LODEL. Il n'est pas necessairement correcte et, surtout, il n'est pas nécessairement fixe, c'est à dire qu'il peut être changer sans préavis par l'herbergeur ou par l'administrateur système. Un changement de ce répertoire rendrait l'ensemble du site inaccessible. En cas de doute utiliser une autre option.
